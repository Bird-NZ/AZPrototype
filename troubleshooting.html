Troubleshooting

Common failure modes across install, auth, design, build, and deployment.

Install and auth issues

  • az prototype --help fails: verify the extension installed correctly and Azure CLI is new enough.
  • GitHub auth errors: run gh auth status. If you chose azure-openai, you can avoid the GitHub auth path.
  • Copilot license failures: use a licensed GitHub account or switch providers for the session.

Stage guard failures

  • No prototype.yaml found: run az prototype init first.
  • Build has weak outputs: rerun design with better artifacts or a tighter context statement.
  • Deploy cannot proceed: run az prototype deploy --dry-run and /preflight to see whether the blocker is infra tooling, subscription state, or missing generated outputs.

When generated architecture looks wrong

  1. Do not hand-patch everything immediately.
  2. Go back to design and improve the requirement signal.
  3. Use template selection and custom agents deliberately.
  4. Rebuild after the design is corrected.

Troubleshooting mindset

The shortest path is usually: verify prerequisites → verify config → dry-run → inspect stage plan → rerun the prior re-entrant stage instead of patching downstream symptoms.
