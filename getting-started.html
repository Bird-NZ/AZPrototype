AZ Prototype Docs

Getting started

Prerequisites, install steps, and first-run checks for AZ Prototype.

Prerequisites

  • Azure CLI 2.50 or newer
  • An Azure subscription you can deploy into
  • gh installed and authenticated if using copilot or github-models
  • A GitHub Copilot Business or Enterprise license if using the copilot provider
  • Terraform or Bicep available when you reach build/deploy for that chosen IaC path

Install

az extension add --name prototype
# later
az extension update --name prototype --allow-preview

The extension auto-installs on first use too, but explicit install makes onboarding cleaner.

Sanity checks

az --version
az account show
gh auth status
az prototype --help

Run these before your first workshop or demo so authentication issues do not eat the whole session.

Choose your AI provider

ProviderBest whenNotes

First-run advice

  • Use a fresh directory for each prototype project.
  • Pick one IaC tool up front so your demos stay consistent.
  • Use preview-friendly language in docs and stakeholder demos because the extension is explicitly in preview.
