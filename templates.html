Templates

Workload templates are optional accelerators for common Azure prototype patterns.

Available templates

TemplateDescriptionTypical services
web-appContainerised web app with SQL backend and APIM gatewayContainer Apps, SQL, Key Vault, APIM
data-pipelineEvent-driven data pipelineFunctions, Cosmos DB, Storage, Event Grid
ai-appAI-powered application starterContainer Apps, Azure OpenAI, Cosmos DB, APIM
microservicesMulti-service architectureContainer Apps, Service Bus, APIM
serverless-apiServerless REST APIFunctions, SQL, Key Vault, APIM

Use a template at init time

az prototype init --name api-demo --location eastus --template serverless-api

What templates do

  • Seed likely service topology
  • Influence build-stage deployment planning
  • Carry configuration defaults and requirements
  • Still allow later design refinements

When not to lean too hard on templates

If your workload is unusual, regulated, or has sharp architectural constraints, treat templates as hints. Let the design stage and custom agents reshape the result.
