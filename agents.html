Customize the system

az prototype agent list
az prototype agent add --file ./my-agent.yaml
az prototype agent override --name cloud-architect --file ./my-architect.yaml
az prototype agent export --name cloud-architect --output ./cloud-architect.yaml

Why this matters

The extension is opinionated: it is not just one model prompt. It is a role-based system with specialization, governance context, and hand-offs. That is why it can cover architecture, IaC, docs, QA, security, and monitoring in a single workflow.

Minimal custom agent YAML

name: my-custom-agent
description: Custom agent for regulated workloads
role: architect
system_prompt: |
  You are a specialized architect for regulated Azure workloads.
constraints:
  - Must use managed identity
  - Must follow naming conventions
tools:
  - terraform
  - bicep
