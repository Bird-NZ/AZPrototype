Quickstart

Copy-paste path from zero to a first generated Azure prototype project.

1. Create a project

mkdir my-prototype && cd my-prototype
az prototype init --name my-prototype --location eastus --template web-app

This creates prototype.yaml and project scaffolding.

2. Generate the design

az prototype design --context "Build a customer-facing web app with API, SQL, Key Vault, and APIM"

Use --artifacts ./requirements when you already have PDFs, docs, slides, screenshots, or notes.

Need examples? This docs site includes a realistic sample pack in sample-artifacts/.

mkdir -p requirements
cp sample-artifacts/* ./requirements/
az prototype design --artifacts ./requirements --context "Use these materials to produce a pragmatic Azure prototype design."
  • 01-executive-brief.md
  • 02-discovery-notes.txt
  • 03-architecture-slides-outline.md
  • 04-sample-issues.csv
  • 05-nonfunctional-requirements.json
  • 06-customer-brief.pdf
  • 07-dashboard-screenshot.svg

3. Build artifacts

az prototype build
# or focus the build
az prototype build --scope infra
az prototype build --scope apps

4. Dry-run deployment before touching Azure

az prototype deploy --dry-run

For Terraform this surfaces a plan. For Bicep it uses What-If.

5. Deploy for real

az prototype deploy

What success looks like

  • prototype.yaml exists
  • Design outputs describe the target architecture
  • Build creates staged infrastructure/app/doc outputs
  • Dry-run shows expected Azure changes
  • Deploy completes stage-by-stage without ad-hoc glue scripts
